Biblioteca digital
Funcionais Não funcionais Inversos

Requisitos Funcionais

  • O sistema deve disponibilizar recursos digitais para facilitar a acessibilidade.
  • O sistema deve apresentar o desempenho de leitura dos estudantes para os professores.
  • O sistema deve fornecer um painel a onde apresente o monitoramento da biblioteca.
  • Os sistema deve apresentar sujestões de livros para os alunos.
  • O painel também deve apresentar a frequência de utilização de plataforma.

Requisitos não Funcionais

  • O sistema deve oferecer o período de uso de livros de 30 dias.
  • Apenas os professores devem recomendar leituras personalizadas.
  • O sistema deverá apresentar os assuntos mais populares com base nos livros mais lidos.
  • O painel dos sistema irá mostrar os livros mais lidos.
  • O painel do sistema irá mostrar os alunos quem mais acessam ao sistema.
  • Apenas alunos e professores podem ter acesso ao sistema.

Requisitos Inversos

  • O sistema não deve permitir que os alunos acessem múltiplos livros ao mesmo tempo sem restrição.
  • O sistema não deve dificultar o processo de reserva de livros, ou seja, não pode ser complicado ou demorado.
  • O sistema não deve permitir o acesso aos livros de forma desorganizada, causando confusão entre os alunos.
  • Evitar que livros sejam extraviados ou devolvidos com atraso, prejudicando outros alunos que precisam daquele material.
  • Os alunos não devem poder acessar os livros recomendados diretamente sem passar pelo processo de autenticação.
  • O sistema não deve permitir a recomendação de livros irrelevantes para os alunos, com base em seu desempenho ou preferências.
  • Evitar que pessoas externas tenham acesso ao sistema, garantindo que apenas alunos e professores da escola possam utilizar os recursos.
  • O sistema não deve deixar de enviar as notificações de devolução, o que poderia causar atrasos e problemas para os alunos.
